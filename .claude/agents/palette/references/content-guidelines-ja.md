# 日本語UIコンテンツガイドライン

> UIテキストの日本語表記ルール。`ux-writing-patterns.md` の汎用パターンを補完する日本語固有の規約。

## 1. 文体ルール

### UIコンテキスト別の文体

| コンテキスト | 文体 | 例 |
|------------|------|-----|
| 操作ラベル（ボタン等） | 常体（動詞終止形 + する） | 保存する、削除する、送信する |
| 説明文・ヘルプ | 敬体（です/ます） | 入力した情報が保存されます |
| エラーメッセージ | 敬体 + 回復手順 | メールアドレスの形式が正しくありません。例: user@example.com |
| 確認ダイアログ | 常体の動詞ベース（タイトル）+ 敬体（説明） | タイトル:「注文を取り消す」説明:「この操作は取り消せません。」|
| プレースホルダー | 常体 | 名前を入力 |
| ツールチップ | 常体 or 敬体（短文） | 新しいタブで開く |
| Toast通知 | 敬体 | 保存しました / 削除に失敗しました |

### 文末の統一

- **同一画面内で常体と敬体を混在させない**（ラベル/説明の文脈切替は許容）
- 体言止めは見出し・ラベルのみ。説明文では使わない

## 2. ボタンラベル規約

### 基本パターン: 「動詞 + する」

| 操作 | ラベル | 避けるべき表現 |
|------|--------|--------------|
| 作成 | 作成する | 新規作成、追加、OK |
| 保存 | 保存する | セーブ、OK、完了 |
| 更新 | 更新する | アップデート |
| 削除 | 削除する | 消す、除去 |
| 送信 | 送信する | 送る、サブミット |
| 検索 | 検索する | 探す、サーチ |
| キャンセル | キャンセル | 取消、やめる、戻る |
| 閉じる | 閉じる | ×（アイコンのみは aria-label 必須） |
| ログイン | ログイン | サインイン（統一する場合はどちらか1つ） |
| ログアウト | ログアウト | サインアウト |

### 具体性のルール

- **抽象的なラベルを避け、操作対象を含める**
- ✅ 「プロフィールを保存する」 → 何を保存するか明確
- ❌ 「OK」「はい」「確認」 → 何が起こるか不明

### アイコン + ラベル

| パターン | 使い分け |
|---------|---------|
| アイコン + ラベル | 標準（推奨） |
| ラベルのみ | テキストで十分に明確 |
| アイコンのみ | スペース制限時。`aria-label` 必須。ツールチップ推奨 |

## 3. エラーメッセージ構造

### 3要素: 何が + なぜ + どうすれば

```
[何が] メールアドレス
[なぜ] の形式が正しくありません。
[どうすれば] 例: user@example.com の形式で入力してください。
```

### エラータイプ別テンプレート

| タイプ | テンプレート | 例 |
|-------|------------|-----|
| 必須未入力 | `{フィールド名}を入力してください` | 氏名を入力してください |
| 形式エラー | `{フィールド名}の形式が正しくありません。{正しい形式}` | メールアドレスの形式が正しくありません。例: user@example.com |
| 範囲外 | `{フィールド名}は{条件}で入力してください` | パスワードは8文字以上で入力してください |
| 重複 | `この{フィールド名}は既に使用されています` | このメールアドレスは既に使用されています |
| サーバーエラー | `{操作}に失敗しました。しばらく時間をおいてもう一度お試しください` | 保存に失敗しました。しばらく時間をおいてもう一度お試しください |
| 権限エラー | `{操作}する権限がありません。管理者にお問い合わせください` | 削除する権限がありません。管理者にお問い合わせください |

### エラー表示の原則

- フィールド直下にインラインで表示（ページ上部のサマリーは補助）
- 赤色 + アイコン（⚠️）で表示（色だけに依存しない）
- エラーのあるフィールドにスクロール + フォーカス

## 4. 空状態メッセージ

### 構成: 状況説明 + 次のアクション

| 空状態 | メッセージ | アクション |
|-------|-----------|----------|
| 初期状態（データなし） | まだ{対象}がありません | [{対象}を作成する] |
| 検索結果なし | 条件に一致する{対象}が見つかりません | [条件をクリア] [新規作成] |
| フィルタ結果なし | 選択した条件では{対象}がありません | [フィルタをクリア] |
| エラーによる空 | {対象}の読み込みに失敗しました | [再読み込み] |

### 表示要素

1. **イラスト/アイコン**（Optional）— 視覚的に空であることを伝える
2. **見出し** — 状況を端的に伝える
3. **説明文** — 次に何をすべきか
4. **アクションボタン** — 最も一般的な次のアクション

## 5. 確認ダイアログ文言

### 構成: タイトル（動詞ベース）+ 説明（影響範囲）+ ボタン（具体的動詞）

```
タイトル: 「{対象}を{動詞}」
説明:     「{影響範囲の説明}。この操作は{取り消し可否}。」
ボタン:   [キャンセル] [{具体的動詞}する]
```

### 破壊的操作の例

```
タイトル: プロジェクト「Alpha」を削除
説明:     関連する12件のタスクと48件のファイルも削除されます。
          この操作は取り消せません。
ボタン:   [キャンセル] [削除する]
```

→ `vision/references/patterns/delete-confirmation.md` 参照

### 非破壊的操作の例

```
タイトル: 変更を保存
説明:     保存すると、現在の設定が全メンバーに反映されます。
ボタン:   [キャンセル] [保存する]
```

## 6. カタカナ外来語

### 内閣告示準拠（長音符号の基本ルール）

**原語の語末が -er, -or, -ar, -y の場合は長音「ー」を付ける:**

| ✅ 正 | ❌ 誤 |
|-------|-------|
| サーバー | サーバ |
| コンピューター | コンピュータ |
| ユーザー | ユーザ |
| プリンター | プリンタ |
| ブラウザー | ブラウザ |
| カレンダー | カレンダ |
| フォルダー | フォルダ |
| マネージャー | マネージャ |

**例外（業界慣行で定着しているもの）:**

| 用語 | 表記 | 理由 |
|------|------|------|
| プログラマ | プログラマ or プログラマー | どちらも許容（プロジェクト内で統一） |

### 一般ルール

- **3音以下のカタカナ語は長音省略可**: アプリ（アプリケーション）
- **省略形は初出時にフルスペルを併記**: API（Application Programming Interface）
- **プロジェクト内で表記を統一** → `terminology-glossary.md` で管理

## 7. 数値・日付フォーマット

### 日付

| フォーマット | 表記 | 使用場面 |
|------------|------|---------|
| 年月日 | 2024/01/15 or 2024年1月15日 | 一般的な日付表示 |
| 年月 | 2024/01 or 2024年1月 | 月単位の表示 |
| 時刻 | 14:30 | 24時間表記 |
| 日時 | 2024/01/15 14:30 | タイムスタンプ |
| 相対 | 3分前、2時間前、昨日 | リアルタイムフィード |

**ルール:**
- 区切りは `/` or 漢字。ハイフン区切り（2024-01-15）はISO形式としてAPIのみ
- 相対表記は48時間以内まで。それ以降は絶対日時
- 曜日を含む場合: 2024/01/15（月）

### 通貨

| フォーマット | 表記 | 例 |
|------------|------|-----|
| 日本円 | ¥{3桁カンマ} | ¥1,234 |
| 日本円（税込） | ¥1,234（税込） | — |
| 日本円（税抜） | ¥1,234（税抜） | — |
| 無料 | 無料 | 「¥0」ではなく「無料」 |

### 数値

| フォーマット | 表記 | 例 |
|------------|------|-----|
| 整数 | 3桁カンマ | 1,234 |
| パーセント | 小数点1桁 | 12.3% |
| 概数 | 単位付き | 1.2万、3.4億 |

## 8. ステータスラベル規約

### ステータスの表記パターン

| カテゴリ | English | 日本語 | 色 |
|---------|---------|--------|-----|
| 成功系 | Active | 有効 | green |
| 成功系 | Complete | 完了 | green |
| 成功系 | Published | 公開中 | green |
| 警告系 | Pending | 保留中 | yellow |
| 警告系 | Draft | 下書き | yellow |
| 警告系 | Expiring | 期限間近 | yellow |
| エラー系 | Error | エラー | red |
| エラー系 | Failed | 失敗 | red |
| エラー系 | Rejected | 却下 | red |
| 中立系 | Inactive | 無効 | gray |
| 中立系 | Archived | アーカイブ済み | gray |
| 情報系 | In Progress | 処理中 | blue |
| 情報系 | Reviewing | レビュー中 | blue |

### ルール

- 状態は「〜中」「〜済み」で動的な意味を表す
- 受動態は避ける（「削除されました」→「削除済み」）
- プロジェクト固有のステータスは `terminology-glossary.md` で定義
