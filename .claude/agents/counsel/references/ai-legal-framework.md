## 1. 日本のAI関連ガイドライン・法規制

### 1.1 内閣府 AI戦略

| 項目 | 内容 |
|------|------|
| AI戦略2022 | AI社会原則の実装推進（人間中心、教育・リテラシー、プライバシー確保、セキュリティ確保、公正競争確保、公平性・説明責任・透明性、イノベーション） |
| 社会原則（7原則） | 人間中心 / 教育・リテラシー / プライバシー / セキュリティ / 公正競争 / 公平性・説明責任・透明性 / イノベーション |
| 法的位置づけ | 法的拘束力なし。ただし行政指導・業界自主規制の根拠となる |

### 1.2 経産省 AIガバナンス

#### AI事業者ガイドライン（第1.1版、2024年4月公表）

| 主体 | 主な義務・推奨事項 |
|------|-------------------|
| AI開発者 | 安全性の確保、公平性への配慮、透明性の確保、アカウンタビリティ |
| AI提供者 | 利用者への適切な情報提供、苦情処理体制の整備、影響評価の実施 |
| AI利用者 | 適切な利用目的での使用、AI出力の検証、人間による最終判断の確保 |

#### AI利用・開発に関する契約チェックリスト（経産省）

- [ ] AI開発契約の類型（探索型/PoC型/開発型）が明確か
- [ ] 学習済みモデルの権利帰属が合意されているか
- [ ] 再利用の範囲（モデル・ノウハウ・データ）が明確か
- [ ] 精度保証の有無と検証方法が定められているか
- [ ] 追加学習・再学習時の条件が整理されているか

### 1.3 総務省 AI利用原則

- AI利用の適正化（利用者のリテラシー向上）
- AI開発の透明性確保
- AI利用における説明可能性
- AIネットワーク化に伴うリスク対応

### 1.4 金融庁 AI・データ活用ガイドライン

| 対象 | 規制内容 |
|------|---------|
| 銀行・保険 | AI利用時のモデルリスク管理、説明責任の確保 |
| 証券 | アルゴリズム取引の管理体制 |
| 全般 | 顧客保護、公正性の確保、外部委託管理 |

## 2. EU AI Act（域外適用を含む）

### 2.1 リスク分類と規制レベル

| リスクレベル | 規制 | 具体例 |
|-------------|------|--------|
| **禁止（Prohibited）** | 全面禁止 | 社会的スコアリング、リアルタイム遠隔生体認証（一部例外あり）、感情認識AI（職場・教育）、脆弱性を悪用するサブリミナル操作 |
| **高リスク（High-risk）** | 適合性評価・登録・監視義務 | 重要インフラ管理AI、教育・採用・信用評価AI、法執行AI、移民管理AI、医療機器AI |
| **限定リスク（Limited risk）** | 透明性義務 | チャットボット（AI使用の開示義務）、ディープフェイク（人工生成の開示義務）、感情認識システム |
| **最小リスク（Minimal risk）** | 義務なし（自主規制推奨） | スパムフィルター、ゲームAI、在庫管理AI |

### 2.2 高リスクAIの義務

| 義務 | 内容 | 日本企業への適用 |
|------|------|----------------|
| リスク管理システム | AI固有リスクの特定・評価・軽減 | EU市場向けサービスに必須 |
| データガバナンス | 学習データの品質・代表性・バイアス管理 | 同上 |
| 技術文書 | システムの設計・開発・テスト記録 | 同上 |
| 記録保持 | 自動ログ記録の義務（最低6ヶ月） | 同上 |
| 透明性 | 利用者への十分な情報提供 | 同上 |
| 人間による監視 | 人間がシステムを監督できる設計 | 同上 |
| 正確性・頑健性 | サイバーセキュリティを含む技術基準 | 同上 |

### 2.3 域外適用（日本企業への影響）

EU AI Actは以下の場合に日本企業にも適用される:

1. **EU域内にAIシステムを上市（placing on the market）する場合**
2. **EU域内の人に対してAIシステムの出力が使用される場合**
3. **EU域外で運用されるAIだが出力がEU域内で利用される場合**

#### 日本企業の実務対応

- [ ] 自社AIサービスのEU市場提供有無を確認
- [ ] EUユーザーからのデータ処理有無を確認
- [ ] リスク分類の自己評価を実施
- [ ] 高リスクに該当する場合は適合性評価の準備
- [ ] EU域内代理人（authorized representative）の選任が必要か検討

## 3. AI生成コンテンツの著作権問題

### 3.1 日本法における整理

#### 著作権法第2条第1項第1号（著作物の定義）

著作物 = 「思想又は感情を創作的に表現したもの」

| 論点 | 現行の整理（文化審議会等） |
|------|------------------------|
| AI単独生成物 | 人の「思想又は感情」の表現ではないため著作物に該当しない |
| AI支援による人の創作 | 人の創作的寄与がある場合は著作物に該当しうる |
| プロンプトの創作性 | 詳細な指示が創作的寄与と認められる可能性はあるが、定型的なプロンプトは不十分 |

#### 著作権法第30条の4（情報解析のための利用）

- AI学習のための著作物の利用は、著作権者の利益を不当に害しない限り許容される
- 「著作権者の利益を不当に害する場合」の判断基準は未確定
- 特定のクリエイターのスタイルを模倣する目的の学習は権利侵害のリスクあり

### 3.2 AI生成コードの著作権リスク

| リスク | 内容 | 対策 |
|--------|------|------|
| 学習データの著作権侵害 | AIが学習した著作物のコードをそのまま出力するリスク | コード類似性チェックツールの利用、コードレビューでの確認 |
| OSSライセンス違反 | AIがOSSコードを学習し、ライセンス表示なしに出力するリスク | SBOM管理、ライセンススキャンの実施 |
| 著作権の不発生 | AI生成コードに著作権が発生しない場合、第三者が自由に利用可能 | 人間による実質的な修正・創作的寄与を加える |
| 営業秘密の混入 | 他社のプロプライエタリコードが学習データに含まれるリスク | AI利用ポリシーでの入力制限、出力の独自性確認 |

### 3.3 実務対応チェックリスト

- [ ] AI生成コードの社内利用ポリシーが策定されているか
- [ ] AI生成コードのライセンススキャンが実施されているか
- [ ] AI生成コードのコードレビュー体制が整備されているか
- [ ] AI生成コードであることの記録（Copilot利用ログ等）が保持されているか
- [ ] 職務著作との関係（従業員がAIを利用して作成したコードの帰属）が整理されているか

## 4. AIシステムにおける個人情報保護（APPI要件）

### 4.1 AI利用時のAPPI固有論点

| 論点 | APPI条文 | 実務要件 |
|------|---------|---------|
| AI学習への個人データ利用 | 第18条（利用目的制限） | 利用目的に「AI/機械学習の改善」が含まれているか |
| プロンプトへの個人情報入力 | 第23条（安全管理措置） | 外部AI APIへの送信が安全管理措置として適切か |
| AI出力に含まれる個人情報 | 第23条 | 出力結果に個人情報が混入するリスクへの対策 |
| プロファイリング | 第17条（利用目的特定） | AIによる自動プロファイリングの利用目的が特定されているか |
| 越境移転（AI API経由） | 第28条（外国第三者提供） | 海外AIプロバイダーへのデータ送信が越境移転に該当するか |

### 4.2 個人情報保護委員会の見解

| テーマ | 見解 |
|--------|------|
| 生成AIと個人情報 | 2023年6月注意喚起: 機密性の高い情報の入力に注意。プライバシーポリシーでのAI利用目的の明示を推奨 |
| プロファイリング | 本人が予測できない態様でのプロファイリングは利用目的の逸脱にあたりうる |
| 越境移転 | クラウドサービス（AI API含む）経由の海外データ処理は越境移転規制の対象となりうる |

### 4.3 AI×個人情報の実務チェックリスト

- [ ] 個人データをAI学習に利用する場合の同意取得手続きが整備されているか
- [ ] 外部AI APIへの個人情報送信の有無が把握されているか
- [ ] 送信する場合、匿名加工・仮名加工の実施状況はどうか
- [ ] AI出力に個人情報が含まれるリスクへの対策があるか
- [ ] AI利用に関するプライバシーポリシーが更新されているか
- [ ] AI APIプロバイダーとのDPA（データ処理契約）が締結されているか

## 5. AI差別・バイアスの法的リスク

### 5.1 法的根拠

| 法律 | 関連条文 | AI固有リスク |
|------|---------|-------------|
| 労働基準法 | 第3条（均等待遇） | 採用AIの性別・人種バイアス |
| 男女雇用機会均等法 | 第5条・第6条 | 採用・配置・昇進AIの性別差別 |
| 障害者差別解消法 | 第8条 | AIサービスの合理的配慮の不提供 |
| 消費者契約法 | 第4条 | AI出力に基づく不当勧誘 |
| 不正競争防止法 | 第2条第1項第20号 | AI生成の品質誤認表示 |

### 5.2 AIバイアスの類型と法的リスク

| バイアス類型 | 説明 | 法的リスク |
|-------------|------|-----------|
| 学習データバイアス | 偏ったデータで学習したモデルの出力偏り | 差別的取扱いによる損害賠償リスク |
| 選択バイアス | 特定グループのデータが過少/過多 | 均等待遇違反（採用・与信分野） |
| 確認バイアス | 既存の偏見を強化する出力 | レピュテーションリスク、行政指導 |
| 代理変数バイアス | 保護属性の代理変数（郵便番号→人種等）による間接差別 | 間接差別として違法となる可能性 |

### 5.3 バイアス対策チェックリスト

- [ ] AIモデルの公平性テスト（demographic parity, equalized odds等）が実施されているか
- [ ] 学習データの偏りが評価されているか
- [ ] 保護属性（性別、年齢、人種、障害等）の直接・間接利用が管理されているか
- [ ] AI意思決定に対する異議申立て手続きが整備されているか
- [ ] 定期的なバイアス監査の体制があるか

## 6. AI透明性・説明可能性要件

### 6.1 日本における要件

| 文書 | 要件内容 |
|------|---------|
| AI事業者ガイドライン | アルゴリズムの概要、学習データの概要、性能・限界の開示を推奨 |
| 金融庁ガイドライン | AIモデルの説明可能性確保、モデルリスク管理の体制整備 |
| 個人情報保護法 | 本人が利用目的を認識できる程度の情報開示（プロファイリング含む） |

### 6.2 EU AI Actにおける透明性義務

| 対象 | 義務内容 |
|------|---------|
| 汎用AIモデル（GPAI） | 技術文書の作成・公開、学習データの著作権遵守方針の公開 |
| 高リスクAI | 利用者が出力を解釈できる説明の提供 |
| チャットボット等 | AI利用であることの開示 |
| ディープフェイク | 人工生成であることの明示 |

### 6.3 説明可能性の実装レベル

| レベル | 内容 | 推奨場面 |
|--------|------|---------|
| L1: 開示 | AIを利用している事実の告知 | 全AIシステム（最低限） |
| L2: 概要説明 | アルゴリズムの種類・学習データの概要 | 消費者向けサービス |
| L3: 判断根拠 | 個別の意思決定の根拠説明（重要因子の提示等） | 採用・与信・保険等 |
| L4: 完全透明性 | モデルの仕様・学習データ・評価結果の完全開示 | 公共利用・規制対象領域 |

## 7. 法令適用判定フロー

```
[AIサービスの法的リスク評価フロー]

1. AIの利用形態を特定
   ├── 自社開発AI
   ├── 外部AI APIの利用
   └── AI搭載SaaSの利用

2. 関連法令の特定
   ├── 著作権法 → セクション3
   ├── 個人情報保護法 → セクション4
   ├── 労働法制（採用AI等） → セクション5
   └── EU AI Act（EU展開の場合） → セクション2

3. リスク分類
   ├── CRIT: 法令違反のおそれ → 即座に是正
   ├── HIGH: ガイドライン非準拠 → 計画的に対応
   ├── MED: ベストプラクティス未実施 → 優先度を判断
   └── LOW: 推奨事項 → 次回見直し時に検討

4. advocate へのハンドオフ
   └── 全判断を判例・行政処分事例で検証依頼
```
