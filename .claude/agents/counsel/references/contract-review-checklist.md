# 契約レビューチェックリスト

> 最終更新: 2026-04-06
> 対象: counsel（法務主査）エージェント
> 用途: SaaS/API契約・OSSライセンス・AI利用契約のリスク評価

---

## 1. SaaS/APIサービス契約レビューポイント

### 1.1 責任制限条項（Limitation of Liability）

| チェック項目 | 確認観点 | リスク判定 |
|-------------|---------|-----------|
| 損害賠償の上限額 | 直近12ヶ月の支払額 or 契約総額が一般的。無制限責任は RED FLAG | HIGH |
| 間接損害の免責 | 逸失利益・データ損失の免責範囲が過度に広くないか | HIGH |
| 除外事項 | 故意・重過失、秘密情報漏洩、知的財産権侵害が除外されているか | MED |
| 補償（Indemnification） | 相互補償か一方的補償か、上限額の有無 | HIGH |

#### RED FLAGS

- [ ] 損害賠償上限がゼロ（ベンダー無責任）
- [ ] 間接損害を含む全損害の免責
- [ ] 補償義務が利用者側のみに課されている
- [ ] 「as-is」提供で一切の保証がない

### 1.2 SLA（Service Level Agreement）

| チェック項目 | 確認観点 | 推奨水準 |
|-------------|---------|---------|
| 稼働率保証 | 月間稼働率の数値（99.9%以上が標準） | 99.9%以上 |
| 計測方法 | 稼働率の計算式が明確か（計画メンテナンス除外等） | 明示必須 |
| SLAクレジット | 未達時の返金・クレジット条件 | 月額10-30% |
| 障害通知義務 | 障害発生から通知までの時間基準 | 1時間以内 |
| 障害復旧目標 | RTO/RPOの定義 | RTO: 4h, RPO: 1h |
| エスカレーション | 重大障害時の連絡体制 | 24/365 |

#### RED FLAGS

- [ ] SLAの数値が定義されていない（「commercially reasonable efforts」のみ）
- [ ] SLA未達時のペナルティがない
- [ ] 計測方法がベンダー側の自己申告のみ

### 1.3 データオーナーシップ・データ取扱い

| チェック項目 | 確認観点 | リスク判定 |
|-------------|---------|-----------|
| データ所有権 | 利用者が投入したデータの所有権が利用者に帰属するか | CRIT |
| データ利用権 | ベンダーがデータを自社目的（統計・改善・AI学習）に利用できるか | HIGH |
| データポータビリティ | 解約時のデータエクスポート手段・期間・形式 | HIGH |
| データ保存場所 | データセンターの所在地（越境移転リスク） | HIGH |
| データ削除 | 解約後のデータ完全削除の保証と期限 | HIGH |
| サブプロセッサー | 再委託先へのデータ共有範囲と通知義務 | MED |

#### RED FLAGS

- [ ] ベンダーがデータを無制限に利用できる条項
- [ ] 解約後のデータ削除義務がない
- [ ] データ保存場所が非開示

### 1.4 契約終了・解約条項

| チェック項目 | 確認観点 | リスク判定 |
|-------------|---------|-----------|
| 解約事由 | 任意解約の可否、解約事由の制限 | HIGH |
| 解約予告期間 | 30日/60日/90日 — 事業影響を考慮 | MED |
| ロックイン条項 | 最低契約期間、自動更新、中途解約ペナルティ | HIGH |
| 移行支援 | 解約時の移行支援義務の有無と内容 | MED |
| 契約存続条項 | 解約後も存続する条項（秘密保持、責任制限等） | LOW |

#### RED FLAGS

- [ ] 解約条項が存在しない
- [ ] 中途解約時に残期間全額の支払義務
- [ ] 自動更新のオプトアウト期間が極端に短い（7日以下）
- [ ] ベンダー側のみ任意解約可能

## 2. OSSライセンス互換性マトリクス

### 2.1 主要ライセンスの分類

| ライセンス | 分類 | コピーレフト | 商用利用 | 再配布要件 |
|-----------|------|------------|---------|-----------|
| MIT | 許容型 | なし | 可 | 著作権表示・ライセンス文の保持 |
| Apache 2.0 | 許容型 | なし | 可 | 著作権表示・ライセンス文・NOTICE・変更点の明示 |
| BSD 2-Clause | 許容型 | なし | 可 | 著作権表示・ライセンス文の保持 |
| BSD 3-Clause | 許容型 | なし | 可 | 著作権表示・ライセンス文・名称利用制限 |
| GPL 2.0 | コピーレフト | 強い | 可 | 派生物も GPL で公開。リンクした全体が対象 |
| GPL 3.0 | コピーレフト | 強い | 可 | GPL 2.0 + 特許ライセンス付与・Tivoization禁止 |
| LGPL 2.1/3.0 | 弱コピーレフト | 弱い | 可 | ライブラリとして動的リンクなら非感染。静的リンクはGPL相当 |
| AGPL 3.0 | ネットワークコピーレフト | 最強 | 可 | ネットワーク経由のサービス提供も派生物扱い |
| MPL 2.0 | ファイルレベルコピーレフト | 中間 | 可 | 変更したファイルのみMPLで公開 |
| Proprietary | 独占 | N/A | 条件次第 | ライセンス条項に完全に従う |

### 2.2 ライセンス互換性マトリクス

```
           MIT   Apache  BSD   GPL2  GPL3  LGPL  AGPL  MPL
MIT         ✓     ✓      ✓     ✓     ✓     ✓     ✓    ✓
Apache 2.0  ✓     ✓      ✓     ✗     ✓     ✓     ✓    ✓
BSD         ✓     ✓      ✓     ✓     ✓     ✓     ✓    ✓
GPL 2.0     ✗     ✗      ✗     ✓     ✗     ✓     ✗    ✗
GPL 3.0     ✗     ✗      ✗     ✗     ✓     ✓     ✓    ✓
LGPL        ✓     ✓      ✓     ✓     ✓     ✓     ✓    ✓
AGPL 3.0    ✗     ✗      ✗     ✗     ✗     ✗     ✓    ✗
MPL 2.0     ✓     ✓      ✓     ✓     ✓     ✓     ✓    ✓

✓ = 行のライセンスのコードを列のライセンスのプロジェクトに含めることが可能
✗ = ライセンス互換性なし（含める場合、列側のライセンスが行側に変更される）
```

### 2.3 SaaS事業でのOSSリスク評価

| リスクレベル | ライセンス | 注意点 |
|-------------|-----------|--------|
| LOW | MIT, BSD, Apache 2.0 | 著作権表示を保持すれば自由に利用可能 |
| MED | LGPL, MPL 2.0 | 動的リンク/ファイル分離で感染を回避可能だが設計に注意 |
| HIGH | GPL 2.0/3.0 | SaaSではネットワーク越し提供は配布に当たらない説が有力だが未確定 |
| CRIT | AGPL 3.0 | ネットワーク越しの提供も配布扱い。ソースコード全体の公開義務 |
| CRIT | 独占ライセンス | 商用利用条件の違反、ライセンス料の未払い |

#### SBOM（Software Bill of Materials）管理

- [ ] プロジェクトのSBOMが生成されているか
- [ ] 全依存パッケージのライセンスが特定されているか
- [ ] CRIT/HIGHライセンスの依存が含まれていないか
- [ ] ライセンス変更の監視体制があるか（dependabot等）

## 3. AI サービス利用契約レビューポイント

### 3.1 学習データ利用条項

| チェック項目 | 確認観点 | リスク判定 |
|-------------|---------|-----------|
| 入力データの学習利用 | プロンプト・入力データがモデル学習に使用されるか | CRIT |
| オプトアウト | 学習利用のオプトアウトが可能か、デフォルトの設定は何か | CRIT |
| APIとWebの差異 | API利用とWeb UI利用で学習ポリシーが異なるか | HIGH |
| データ保持期間 | 入力データの保持期間（30日、無期限等） | HIGH |
| Zero Data Retention | ZDR（即時削除）オプションの有無 | MED |

#### 主要AIプロバイダーの学習ポリシー（2026年4月時点、要定期確認）

| プロバイダー | API | Web UI | オプトアウト |
|-------------|-----|--------|------------|
| OpenAI | デフォルト非学習 | デフォルト学習 | API: 不要 / Web: 設定で可 |
| Anthropic | デフォルト非学習 | 利用規約による | API: 不要 |
| Google (Gemini) | 利用規約による | デフォルト学習 | 設定で可 |

### 3.2 出力物の権利帰属

| チェック項目 | 確認観点 | リスク判定 |
|-------------|---------|-----------|
| 出力物の著作権 | AI生成物に著作権が発生するか（日本法: 人の創作的寄与が必要） | HIGH |
| 利用権 | 出力物の商用利用・再配布が許可されているか | HIGH |
| 類似出力リスク | 他ユーザーに類似の出力が生成される可能性への対処 | MED |
| 第三者権利侵害 | AI出力が第三者の著作権を侵害するリスクと責任の所在 | HIGH |

### 3.3 データセキュリティ・コンプライアンス

| チェック項目 | 確認観点 | リスク判定 |
|-------------|---------|-----------|
| データ処理場所 | APIリクエストの処理リージョン | HIGH |
| SOC 2 / ISO 27001 | セキュリティ認証の取得状況 | MED |
| DPA（Data Processing Agreement） | データ処理契約の締結可否 | HIGH |
| サブプロセッサー | 下請けプロセッサーのリストと通知義務 | MED |
| インシデント通知 | セキュリティインシデント時の通知義務と期限 | HIGH |

## 4. 契約上のRED FLAGSチェックリスト（全契約共通）

以下のいずれかに該当する場合、即座にエスカレーションする。

### 4.1 CRIT — 契約締結前に必ず是正が必要

- [ ] **無制限の責任**: 損害賠償の上限が設定されていない
- [ ] **解約条項の不在**: 契約解除の方法が規定されていない
- [ ] **専属管轄の不利益**: 紛争解決が相手方の国/地域の裁判所に専属管轄
- [ ] **データ所有権の喪失**: 投入データの所有権がベンダーに移転する条項
- [ ] **無断のデータ学習利用**: 入力データがAI学習に使用され、オプトアウト不可
- [ ] **一方的変更権**: ベンダーが事前通知なく契約条件を変更できる

### 4.2 HIGH — 交渉で改善すべき

- [ ] **過度な補償義務**: 利用者側のみに無制限の補償義務
- [ ] **競業避止条項**: 競合サービスの利用を制限する条項
- [ ] **知的財産権の移転**: 利用者が作成したカスタマイズ・連携の権利がベンダーに帰属
- [ ] **監査権の不在**: ベンダーのセキュリティ・コンプライアンスを監査する権利がない
- [ ] **解約後のデータ返却義務なし**: 解約後にデータがどうなるか不明

### 4.3 MED — 認識の上で許容可能

- [ ] **準拠法が外国法**: 日本法以外が準拠法（ただし仲裁条項があれば軽減可）
- [ ] **自動更新**: 自動更新条項（オプトアウト期間が30日以上なら許容範囲）
- [ ] **利用制限条項**: Fair use policy等の曖昧な利用制限

## 5. 契約レビュープロセス

### 5.1 レビュー手順

```
1. 契約類型の特定
   ├── SaaS/クラウドサービス → セクション1を適用
   ├── OSSライブラリ → セクション2を適用
   └── AI/MLサービス → セクション3を適用
   
2. RED FLAGSチェック（セクション4）
   ├── CRIT該当あり → 即座にエスカレーション
   ├── HIGH該当あり → 交渉ポイントとして記録
   └── MED以下 → リスク認識の上で許容判断
   
3. データ関連条項の精査
   ├── 所有権・利用権
   ├── 保存場所・越境移転
   └── 削除・返却条件
   
4. 責任・SLA条項の精査
   ├── 損害賠償の上限
   ├── SLA数値と未達ペナルティ
   └── 免責事項の範囲
   
5. 契約終了条項の精査
   ├── 解約条件・予告期間
   ├── 移行支援の有無
   └── 契約存続条項
```
