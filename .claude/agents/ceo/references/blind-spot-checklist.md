# Blind Spot Checklist

CEOエージェントおよびAudit Keijiが全判断に適用する盲点チェックリスト。
判断の質を構造的に担保するための検出パターン集。

---

## 使用タイミング

- Four Minds プロトコルの Audit Keiji 評価時（必須）
- 重要な判断の最終チェック時
- 定期監査（月次）時

---

## 盲点パターン一覧

### 1. Goal Stacking（目的の積み重ね）

**検出条件**: 1つの施策に3つ以上の独立した目的がある

**リスク**: 優先順位が曖昧になり、どの目的も中途半端に終わる

**対処**: 施策を分離する。1施策1目的が原則

```
検出例:
  施策: 「新機能Xを追加する」
  目的1: 新規ユーザー獲得
  目的2: 既存ユーザーの継続率向上
  目的3: ブランドイメージ刷新
  → 3つの独立した目的 → 分離を提案
```

### 2. Conditional Drift（条件付き承認の漂流）

**検出条件**: 条件付きGoに期限が設定されていない

**リスク**: 条件が永遠に満たされないまま施策が進行する

**対処**: 全ての条件付き承認に2週間の期限を設定する。期限到来時に再判断

```
検出例:
  判断: 「データが揃ったらGo」
  期限: なし
  → 2週間期限を付与: 「2週間以内にデータが揃わなければ No-Go に変更」
```

### 3. Small-N Trap（少数サンプルの罠）

**検出条件**: N<30のデータに基づいて判断を下そうとしている

**リスク**: 統計的に信頼できない結論を根拠にする

**対処**: 信頼区間を明示する。N<30の場合は「参考値」として扱い、判断の根拠にしない

```
検出例:
  主張: 「新機能のCVRが15%向上した」
  データ: N=12
  → 信頼区間を計算: 95%CI [3.2%, 26.8%] → 「改善なし」もあり得る
  → 判断の根拠にしない。サンプル数を増やしてから再評価
```

### 4. Happy Path Bias（楽観シナリオ偏重）

**検出条件**: ロールバック計画がない、または失敗シナリオが検討されていない

**リスク**: 失敗時に不可逆な損害が発生する

**対処**: ロールバック計画なしの場合は即 No-Go。例外なし

```
検出例:
  提案: 「新決済プロバイダに移行する」
  ロールバック計画: なし
  → 即 No-Go
  → 「旧プロバイダとの並行稼働 + 段階的移行」の代替案を要求
```

### 5. Decision Duplication（判断の繰り返し）

**検出条件**: 同一テーマについて3回以上判断が繰り返されている

**リスク**: 判断がループし、時間を浪費する。過去の判断が活かされていない

**対処**: 過去セッションの判断ログを参照し、新しい情報がない場合は前回の判断を維持

```
検出例:
  テーマ: 「グローバル展開の優先市場」
  過去判断: 3回（2026-01, 2026-02, 2026-03）
  新情報: なし
  → 前回の判断を維持。新しいデータが出るまで再議論しない
```

### 6. Audit Warning Neglect（監査警告の放置）

**検出条件**: 過去のAudit警告に対して「認識した」が「対策した」の記録がない

**リスク**: リスクが認識されたまま放置され、顕在化する

**対処**: 全Audit警告にステータス追跡を付与。「認識」→「対策立案」→「対策実行」→「効果確認」

```
検出例:
  警告: 「モデレーション体制が拡大速度に追いつかない可能性」
  ステータス: 「認識した」（3週間前）
  対策実行: なし
  → エスカレーション: 対策立案の期限を設定
```

### 7. Team Proposal Auto-Approval（チーム提案の自動承認）

**検出条件**: チームからの提案の承認率が95%を超えている

**リスク**: 質的フィルタリングが機能していない

**対処**: 提案の却下理由を構造化し、承認基準を明確にする

```
検出例:
  承認率: 98.4%（過去3ヶ月）
  → フィルタリング基準を見直す
  → 各提案にFour Minds評価を義務付ける
```

### 8. Marketing Depth Gap（マーケティング判断の深度不足）

**検出条件**: 技術・プロダクトには構造的分析があるが、マーケティングには感覚的判断しかない

**リスク**: マーケティング投資の効果が検証されず、無駄になる

**対処**: マーケティング施策にも同じ判断基準（仮説→検証→データ→判断）を適用

```
検出例:
  技術判断: 「N=500で統計的に有意なA/Bテスト結果に基づきGo」
  マーケ判断: 「SNSの反応が良さそうだからGo」
  → マーケ判断にもデータ根拠を要求
```

### 9. Delegation Completion Gap（委譲後のフォローアップ不足）

**検出条件**: タスクが委譲されたが、完了確認が行われていない

**リスク**: 「委譲 = 完了」と誤認し、実際には未完了のまま放置される

**対処**: 委譲時に完了基準と確認タイミングを設定する

```
検出例:
  委譲: 「山口さんにPO引継ぎ」
  完了確認: なし
  → 完了基準を明示: 「引継ぎドキュメント完成 + 1週間の独立運用確認」
  → 確認タイミング: 2週間後
```

### 10. Long-term Strategy Staleness（長期戦略の陳腐化）

**検出条件**: 長期戦略の前提が3ヶ月以上再検証されていない

**リスク**: 前提が崩壊したまま戦略が実行され、リソースが浪費される

**対処**: 四半期ごとに長期戦略の前提条件を再検証する

```
検出例:
  戦略: 「2027年4月に韓国展開」
  前提: 「Luna+が2026年10月にリリース済み」
  最終検証: 2026-01（3ヶ月前）
  → 前提の現状を再確認: Luna+のスケジュールは計画通りか?
```
