# Onboarding Flow Pattern

## Overview

| 項目 | 値 |
|------|-----|
| Pattern | Onboarding Flow |
| Category | Onboarding / Activation |
| Complexity | High |
| Reference | SmartHR「初期設定ウィザード」・Notion「Getting Started」相当 |

**解決する課題:** 新規ユーザーがサービスの価値を早期に体験できるよう、初期設定・主要機能のディスカバリーをスムーズにガイドする。離脱率を下げ、アクティベーション率を上げる。

**使用場面:**
- 新規登録直後の初期設定（プロフィール・環境設定等）
- 新機能のチュートリアル
- 空の状態からの最初のコンテンツ作成ガイド

## Interaction Flow

### Type 1（ウィザード）フロー

```
新規登録完了
  └→ セットアップウィザード開始
      ├→ ステップ完了 → 次ステップへ
      ├→ スキップ → 次ステップへ（後から設定可能なステップのみ）
      ├→ 戻る → 前ステップへ（入力は保持）
      └→ 全ステップ完了 / 全スキップ
           └→ ダッシュボードへ遷移
               └→ 完了数に応じたウェルカムメッセージ表示
```

### Type 2（チェックリスト）フロー

```
ダッシュボード表示
  └→ チェックリストカード表示（全完了まで）
      └→ アイテムをクリック
           └→ 該当機能ページへ誘導 or インライン実行
               └→ 完了 → チェックマーク + アニメーション
                   └→ 全5件完了 → カードを折りたたみ（削除しない）
```

### スキップ戦略

| ステップ | スキップ可否 | 理由 |
|---------|------------|------|
| プロフィール基本情報 | 可（後から設定） | 任意情報 |
| メール確認 | 不可 | セキュリティ必須 |
| 支払い情報（無料期間中） | 可 | 強制はCVRを下げる |
| チュートリアル | 可 | 経験者UX配慮 |

## Progress Indication

### ウィザードプログレスバー

```
○─────○─────●─────○
1      2     3     4
完了   完了  現在  未実施

● = 現在（Brand/600 塗り）
✓ = 完了（Green/500 塗り）
○ = 未実施（Black/200 塗り）
```

| Property | Value |
|----------|-------|
| ステップドット径 | 12px |
| ライン高 | 2px |
| 完了色 | Green/500 `#18CF83` |
| 現在色 | Brand/600 `#5538EE` |
| 未実施色 | Black/200 `#DADADD` |

## Accessibility

- ウィザードステップ: `aria-current="step"` で現在ステップをマーク
- プログレス: `role="progressbar"` + `aria-valuenow` + `aria-valuemax`
- スキップリンク: キーボードユーザーがチュートリアルをスキップできる仕組み
- Type 3ツールチップ: `aria-describedby` + `role="tooltip"`

## Do / Don't

### Do
- ✅ 必須ステップ数を最小化する → 3〜5ステップが上限
- ✅ 各ステップで「何のためにこの情報が必要か」を説明する
- ✅ 入力済みデータは戻るで保持する → 再入力の手間を省く
- ✅ チェックリストは完了後も参照できる状態に保つ

### Don't
- ❌ 全ての初期設定を必須にしない → 任意情報のスキップを許容
- ❌ プログレスを見せずに長いウィザードを続けない → 残りステップ数を常に表示
- ❌ 登録直後に課金情報を強制収集しない → 価値提供後に促す
